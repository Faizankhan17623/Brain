# Brain — Personal AI Wiki Template

A personal knowledge wiki built on the **Karpathy LLM-wiki pattern**: an AI (Claude or ChatGPT) writes and maintains all wiki content; you curate sources, ask questions, and direct analysis. Knowledge compounds over time — not re-derived on every query.

**UI:** Obsidian &nbsp;|&nbsp; **AI writer:** Claude Code (or ChatGPT) &nbsp;|&nbsp; **Auto-sync:** your chat history lands in `00-inbox/` automatically

---

## Quick Start (for anyone forking this)

### 1. Clone & open in Obsidian

```bash
git clone https://github.com/Faizankhan17623/Brain.git
cd Brain
```

Open the `Brain/` folder as an Obsidian vault.

### 2. Set up your environment

```bash
cp .env.example .env
```

Edit `.env` and paste your session key (see instructions below for Claude or ChatGPT).

### 3. Install dependencies

```bash
pip install requests python-dotenv
```

### 4. Test the sync

```bash
python sync_claude.py      # if you use Claude
# python sync_chatgpt.py   # if you use ChatGPT (see below)
```

New conversations will appear in `00-inbox/`.

### 5. Set up auto-sync (Windows Task Scheduler)

Run every 5 hours automatically:

```
Action: python C:\path\to\Brain\sync_claude.py
Trigger: Daily, repeat every 5 hours
```

## Getting Your Session Key

### Claude (claude.ai)

1. Open [claude.ai](https://claude.ai) and log in
2. Press `F12` → **Application** tab → **Cookies** → `claude.ai`
3. Find the cookie named `sessionKey` — copy its value
4. Paste into `.env` as `CLAUDE_SESSION_KEY=...`

> Session keys expire when you log out. Re-copy if sync stops working.

### ChatGPT (chat.openai.com)

1. Open [chat.openai.com](https://chat.openai.com) and log in
2. Press `F12` → **Application** tab → **Cookies** → `chat.openai.com`
3. Find the cookie named `__Secure-next-auth.session-token` — copy its value
4. Paste into `.env` as `CHATGPT_SESSION_TOKEN=...`

> ChatGPT does not have an official conversation export API. `sync_chatgpt.py` uses the internal `/backend-api/conversations` endpoint — it may break if OpenAI changes their API.

## Personalizing Your Vault

After forking, update these two files with your own details:

**`CLAUDE.md`** — tells the AI who you are and what your vault covers. Edit the "About" section with your name, role, and goals. This is what Claude reads at the start of every session.

**`🏠 Home.md`** — your vault dashboard. Update the quick links and stats to match your folders.

**`index.md`** — master catalog. Claude keeps this updated automatically as you ingest sources.

Everything else (daily notes, tracker pages, topic pages) is generated by Claude as you use the vault.

## How It Works

```
Your chats → sync_*.py → 00-inbox/ → /inbox command → wiki pages → index.md
```

| Operation | What happens |
|-----------|-------------|
| `/inbox` | Claude reads a source, writes a summary page, updates index.md and log.md |
| `/daily` | Claude creates today's daily note with Yesterday/Tomorrow chain |
| Shortcut commands | Natural language commands — see CLAUDE.md for the full list |

A single `/inbox` ingest typically touches 5–15 wiki pages.

## Vault Structure

```
Brain/
├── 00-inbox/        New synced chats land here (auto-sync output)
├── 01-daily/        YYYY-MM-DD daily notes, chained Yesterday/Tomorrow
├── 02-projects/     One sub-folder per active project
├── 03-meetings/     Meeting notes
├── 04-resources/    Reference pages, learnings, concept wiki pages
├── 05-goals/        Annual + quarterly goals
├── 06-people/       CRM-style notes on contacts and recruiters
├── 07-archive/      Never delete — move here when done
│
├── Projects/        Hub pages per project
├── Topics/          Topic wiki pages (AI Engineering, DSA, etc.)
├── Companies/       Target employer pages
├── People/          Personal CRM pages
│
├── sync_claude.py   Pulls new claude.ai chats → 00-inbox/
├── build_vault_v2.py  One-time rebuild of Conversations/ from JSON export
├── index.md         Master catalog of all wiki pages
├── log.md           Append-only activity log
└── 🏠 Home.md       Vault dashboard
```

## What's Excluded from Git (Privacy)

```
.env                  # your session keys
conversations.json    # raw conversation export (can be 30MB+)
Conversations/        # private conversation archive
00-inbox/             # private synced chats
sync_state.json       # sync metadata
memories.json         # private AI memory data
```

These are in `.gitignore`. Never commit `.env` or session keys.
